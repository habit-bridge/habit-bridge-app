# Habit Bridge Demo — 화면별 UI 요소 명세 (MVP)

본 문서는 [요구사항](./requirements.md)과 [화면 이동 흐름](./screen-flow.md)을 바탕으로, **각 화면에 어떤 UI 요소가 들어가야 하는지**를 정리한다. 디자인 시안 이전 단계의 **컴포넌트/콘텐츠 명세**이며, 실제 디자인에서는 컴포넌트의 시각적 표현이 달라질 수 있다.

---

## 0. 공통 디자인 토큰 / 컴포넌트

### 0.1 공통 레이아웃

| 영역 | 설명 |
|------|------|
| **Status Bar** | OS 상태 표시(배터리/시간) — 시스템 영역 |
| **App Bar (Top)** | 좌측: 뒤로가기 또는 타이틀 / 중앙: 화면 타이틀 / 우측: 액션(예: `+`, 저장) |
| **Content** | 화면 본문(스크롤 영역) |
| **Bottom Tab Bar** | 4개 탭: 홈 / 내 참여 / 랭킹 / 마이 (S10/S30/S50/S60에만 노출) |
| **Floating/Sticky CTA** | 일부 화면 하단 고정 버튼(예: 참여하기) |

### 0.2 공통 컴포넌트

- **PrimaryButton** — 메인 액션(가입/로그인/참여/업로드 등)
- **SecondaryButton / TextButton** — 보조 액션
- **InputField** — 라벨 + 입력 + 인라인 에러 메시지
- **DatePicker / NumberStepper** — 날짜·기간 입력
- **Card** — 챌린지/참여/랭킹 아이템 단위
- **StatusBadge** — 상태 표기(`진행 중`, `성공`, `실패`, `대기`)
- **ProgressBar** — 인증 진행률
- **EmptyState** — 빈 목록 안내 + 액션 CTA
- **Toast / Snackbar** — 일시 알림(성공/실패)
- **Dialog / BottomSheet** — 확인 다이얼로그, 보증금 안내
- **LoadingSpinner / SkeletonLoader** — 로딩 인디케이터
- **Avatar** — 사용자 표시(이니셜 또는 이미지)

### 0.3 공통 상태 UI 패턴

| 상태 | UI 표현 |
|------|---------|
| Loading | Skeleton(목록) / Spinner(액션) |
| Empty | EmptyState 컴포넌트 + CTA |
| Error | 인라인 에러 텍스트 + Retry 버튼 / Toast |
| Success | Toast 1.5~2초 |

## 2. S01 로그인

**목적**: 이메일/비밀번호 로그인 + 회원가입 화면 이동.

| 영역 | 요소 |
|------|------|
| App Bar | 타이틀: "로그인" (또는 로고만) |
| 본문 | 1) 로고 / 헤드라인<br>2) **InputField: 이메일** (자동완성, 키보드 이메일 모드)<br>3) **InputField: 비밀번호** (마스킹, 표시 토글)<br>4) 인라인 에러 메시지("이메일 또는 비밀번호가 올바르지 않습니다")<br>5) **PrimaryButton: "로그인"** (로딩 상태 지원)<br>6) **TextButton: "회원가입"** → S02 |

검증:

- 이메일 형식 검증(클라이언트)
- 비밀번호 최소 길이(예: 8자) 안내

## 4. S10 홈 / 챌린지 목록

**목적**: 종료되지 않은 챌린지를 탐색하고 참여 진입.

| 영역 | 요소 |
|------|------|
| App Bar | 타이틀: "Habit Bridge" / 우측 액션: **+ (새 챌린지)** → S12 |
| 본문 | 1) (선택) 검색 바<br>2) 필터/정렬 — `진행 중` / `시작 예정` (Chip)<br>3) **챌린지 카드 리스트** |
| Bottom Tab | 홈(active) / 내 참여 / 랭킹 / 마이 |

**챌린지 카드(Card)** 요소:

- 제목(굵게)
- 소개 1~2줄(말줄임)
- StatusBadge: `시작 예정` / `진행 중`
- 메타 라인: 시작일 ~ 종료일, 인증 주기, **보증금(XRP)**
- 참여 인원(있다면)
- 우측 화살표(상세 진입 표시)

빈 상태:

- "지금 참여 가능한 챌린지가 없습니다" + `새 챌린지 만들기` CTA → S12

## 5. S11 챌린지 상세

**목적**: 챌린지 정보 확인 + 참여 진입.

| 영역 | 요소 |
|------|------|
| App Bar | 좌측 뒤로가기 / 타이틀: 챌린지 제목 / 우측: 공유 아이콘(선택) |
| 본문 | 1) **헤더 섹션**: 제목, StatusBadge<br>2) **메타 카드**: 시작일/종료일, 기간(주), 인증 주기, 보증금(XRP)<br>3) **소개** 섹션 (멀티라인 텍스트)<br>4) **인증 방법 설명** 섹션<br>5) **참여 정보** — 현재 참여 인원, 내가 이미 참여 중인지 여부 표시<br>6) (선택) 챌린지 생성자 정보 |
| Sticky 하단 | **PrimaryButton: "참여하기"** → S20 |

버튼 상태:

| 조건 | 버튼 상태 / 라벨 |
|------|--------------------|
| 종료된 챌린지 | 비활성, "종료된 챌린지" |
| 이미 참여 중 | "내 참여 보기" → S31 |
| 미참여 + 참여 가능 | "참여하기" (활성) |

## 6. S12 챌린지 생성

**목적**: 새 챌린지 생성 폼.

| 영역 | 요소 |
|------|------|
| App Bar | 좌측 닫기(X) / 타이틀: "새 챌린지" / 우측: TextButton "저장" (선택) |
| 본문 (Form) | 1) **InputField: 제목** (필수)<br>2) **TextArea: 소개** (멀티라인)<br>3) **DatePicker: 시작일** (오늘 이후만 선택 가능)<br>4) **NumberStepper: 기간(주)** (예: 1~12)<br>5) **Selector: 인증 주기** — `매일` / `주 1회` / `주 3회` 등<br>6) **TextArea: 인증 방법 설명**<br>7) **InputField: 보증금(XRP)** — 숫자 키패드, decimal 허용<br>8) **요약 카드** — 자동 계산된 종료일 미리보기 |
| Sticky 하단 | **PrimaryButton: "챌린지 만들기"** |

검증:

- 필수 항목 누락 → 인라인 에러
- 보증금 최소값(예: 1 XRP) 정책 표시
- 종료일 = 시작일 + 기간(주) 자동 계산하여 폼 상단/하단에 미리보기

## 7. S20 참여 확인 (보증금 안내)

**목적**: 보증금/에스크로 정책 안내 후 서명 시작.

| 영역 | 요소 |
|------|------|
| App Bar | 좌측 뒤로가기 / 타이틀: "참여 확인" |
| 본문 | 1) **챌린지 요약 카드** — 제목, 기간, 인증 주기<br>2) **보증금 강조 박스** — `XX XRP`를 에스크로로 예치한다는 안내<br>3) **안내 텍스트**<br>&nbsp;&nbsp;&nbsp;- 성공 시: "당신의 이름으로 기부됩니다"<br>&nbsp;&nbsp;&nbsp;- 실패 시: "서비스 이름으로 기부됩니다"<br>4) **체크박스**: "내용을 확인했습니다" (활성화 게이트)<br>5) **외부 지갑(Xaman) 안내** — 아이콘 + 짧은 설명 |
| Sticky 하단 | **PrimaryButton: "Xaman으로 서명하기"** (체크 시 활성) |

비고:

- 버튼 누르면 서버에서 EscrowCreate payload를 생성하고 **S21**(외부)로 이동.
- 진행 중에는 버튼이 로딩 상태가 되어 중복 호출 방지.

## 8. S21 Xaman 서명 (외부)

**목적**: 외부 지갑 앱(또는 외부 브라우저의 Xaman 페이지)에서 서명.

자체 UI는 없음. 다만 앱 측에서 제공해야 하는 보조 요소:

| 영역 | 요소 |
|------|------|
| (오버레이) | 1) "Xaman 앱이 열렸습니다" 안내 카드<br>2) **TextButton: "이미 서명했어요"** → S22로 진입<br>3) **TextButton: "취소"** → S20 복귀 |

비고:

- Xaman 앱이 설치되지 않은 사용자를 위한 **설치 안내 링크**도 노출.
- 딥링크 복귀 시 자동으로 S22로 라우팅된다.

## 9. S22 서명 결과 처리 (Pending)

**목적**: 서버에서 서명 결과를 검증/확인할 때까지의 대기 화면.

| 영역 | 요소 |
|------|------|
| App Bar | (뒤로가기 비활성 또는 "백그라운드에서 계속" 안내) |
| 본문 | 1) **대형 LoadingSpinner / 애니메이션**<br>2) 안내 텍스트: "서명을 확인하고 있어요"<br>3) **단계 인디케이터**: `서명 수신 → 트랜잭션 확인 → 참여 활성화`<br>4) 부가 텍스트: "최대 1분 정도 걸릴 수 있어요"<br>5) **TextButton: "백그라운드에서 계속 진행"** → S30로 이동 |

결과 분기:

- 성공(`ACTIVE`) → **S31 참여 상세**로 자동 이동, Toast: "참여가 시작되었어요!"
- 실패/거절 → **S20 또는 S11**로 복귀, 사유 안내

## 10. S30 내 참여 목록

**목적**: 내가 참여 중이거나 종료된 챌린지 목록.

| 영역 | 요소 |
|------|------|
| App Bar | 타이틀: "내 참여" |
| 본문 | 1) **세그먼트 컨트롤 / 탭**: `진행 중` / `종료` / `대기 중`<br>2) **참여 카드 리스트** |
| Bottom Tab | 내 참여(active) |

**참여 카드** 요소:

- 챌린지 제목
- StatusBadge: `대기` / `진행 중` / `성공` / `실패` / `취소`
- 진행 중인 경우:
  - **ProgressBar**: 완료한 인증 수 / 전체 인증 수
  - "다음 인증 마감: D-2" 같은 보조 텍스트
- 종료된 경우:
  - 결과 라벨 + 보증금 + 기부 귀속 미리보기
- 우측 화살표 → S31(진행 중/대기) 또는 S40(종료)

빈 상태:

- "참여 중인 챌린지가 없어요" + `챌린지 둘러보기` CTA → S10

## 11. S31 참여 상세 / 인증 슬롯

**목적**: 한 참여의 인증 진행 상황 확인 + 업로드 진입.

| 영역 | 요소 |
|------|------|
| App Bar | 좌측 뒤로가기 / 타이틀: 챌린지 제목 |
| 본문 | 1) **요약 카드** — StatusBadge, 시작/종료일, 보증금<br>2) **진행률 섹션**<br>&nbsp;&nbsp;&nbsp;- ProgressBar (`3 / 8 인증 완료`)<br>&nbsp;&nbsp;&nbsp;- 남은 인증 수 / 남은 일수<br>3) **현재 주기 카드** — "이번 주기에 인증할 수 있어요" / "이번 주기는 인증 완료" / "마감되었습니다"<br>4) **인증 슬롯 리스트** — 주차별/일자별 슬롯 카드<br>&nbsp;&nbsp;&nbsp;- 슬롯 상태: `완료(✓)` / `대기` / `마감(놓침)`<br>&nbsp;&nbsp;&nbsp;- 완료된 슬롯은 업로드한 썸네일 미리보기<br>5) (종료된 경우) **결과 보러 가기** 카드 → S40 |
| Sticky 하단 | **PrimaryButton: "이번 주기 인증하기"** → S32 |

버튼 상태:

| 조건 | 버튼 상태 |
|------|-----------|
| 인증 가능 슬롯 있음 | 활성, "이번 주기 인증하기" |
| 이미 완료 | 비활성, "이번 주기 인증 완료" |
| 마감 또는 챌린지 종료 | 비활성, "인증 마감" |

## 12. S32 사진 인증 업로드

**목적**: 날짜가 포함된 사진을 업로드.

| 영역 | 요소 |
|------|------|
| App Bar | 좌측 닫기(X) / 타이틀: "사진 인증" |
| 본문 | 1) **귀속 슬롯 안내**: "N주차 / YYYY-MM-DD 슬롯에 업로드됩니다"<br>2) **사진 미리보기 영역** (선택 전: 빈 영역 + 아이콘)<br>3) **버튼 그룹**<br>&nbsp;&nbsp;&nbsp;- **PrimaryButton: "카메라로 촬영"**<br>&nbsp;&nbsp;&nbsp;- **SecondaryButton: "갤러리에서 선택"**<br>4) **체크리스트 안내**<br>&nbsp;&nbsp;&nbsp;- "사진에 오늘 날짜가 보이도록 찍어주세요"<br>&nbsp;&nbsp;&nbsp;- 허용 파일 타입/용량 제한 안내<br>5) (선택) 캡션 InputField |

검증/에러:

- 슬롯 만료 → 다이얼로그로 안내 후 S31 복귀

성공 시:

- Toast: "인증이 등록되었어요"
- S31로 복귀하여 슬롯 상태 즉시 갱신

## 13. S40 결과 / 정산 상세

| 영역 | 요소 |
|------|------|
| App Bar | 좌측 뒤로가기 / 타이틀: "결과" |
| 본문 | 1) **결과 히어로 영역**<br>&nbsp;&nbsp;&nbsp;- 큰 StatusBadge: `성공` / `실패`<br>&nbsp;&nbsp;&nbsp;- 챌린지 제목<br>&nbsp;&nbsp;&nbsp;- 기간 요약<br>2) **인증 요약 카드**<br>&nbsp;&nbsp;&nbsp;- 완료 / 전체 인증 수<br>&nbsp;&nbsp;&nbsp;- ProgressBar<br>3) **기부 귀속 카드**<br>&nbsp;&nbsp;&nbsp;- "OO 님의 이름으로 기부되었어요" 또는 "Habit Bridge 서비스 이름으로 기부되었어요"<br>&nbsp;&nbsp;&nbsp;- 보증금 금액(XRP)<br>4) (후순위) **온체인 트랜잭션 링크** — 익스플로러 이동<br>5) **원본 챌린지 보기** TextButton → S11 |
| Sticky 하단 | (선택) **PrimaryButton: "공유하기"** |

비고:

- 절대 변경되지 않는 화면이며, 같은 결과는 항상 같은 표시를 가져야 한다(멱등).

## 14. S50 랭킹

**목적**: 두 가지 TOP 10 랭킹 표시.

| 영역 | 요소 |
|------|------|
| App Bar | 타이틀: "랭킹" |

**랭킹 항목 Row** 요소:

- 순위 숫자(1~10) — 1~3위는 강조(아이콘/색상)
- Avatar
- displayName
- 우측 수치:
  - Donation 탭: 누적 기부액(XRP)
  - Success 탭: 성공 횟수

빈 상태:

- "아직 집계된 랭킹이 없어요"

비고:

- 1~3위는 시각적으로 강조(메달 아이콘 등)하면 사용자 인지가 좋아짐.

## 17. 접근성 / UX 체크리스트

- 모든 인터랙티브 요소는 **최소 44x44pt 터치 타깃**.
- 에러 메시지는 **빨간색 + 아이콘** 등 색상에만 의존하지 않게 표현.
- 보증금/XRP 금액은 **소수점 자리수**를 통일(예: 2~6자리).
- 외부 앱 전환(S21) 시 **앱 복귀 가이드**를 명확히 노출.
- 로딩 중에는 **PrimaryButton의 중복 클릭 방지** 처리 필수(특히 S20, S32).
- 모든 리스트 화면(S10, S30, S50)은 **Pull-to-refresh** 지원 권장.
